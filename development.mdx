<Info>
  **Prerrequisitos**:
  - Acceso de administrador a OCLÜ CRM
  - Conocimientos básicos de configuración de sistemas
  - Acceso a la documentación técnica
</Info>

Esta guía está dirigida a desarrolladores y administradores técnicos que necesitan personalizar o integrar OCLÜ CRM.

<Steps>
<Step title="Acceso de Desarrollador">

Para acceder a las funciones de desarrollo:

1. Solicita credenciales de desarrollador a soporte@oclucrm.com
2. Accede al panel de administración avanzado
3. Configura tu entorno de desarrollo local

```bash
# Configuración básica del entorno
export OCLU_API_KEY="tu_api_key"
export OCLU_ENVIRONMENT="development"
```

</Step>

<Step title="API y Integraciones">

OCLÜ CRM ofrece API REST para integraciones:

```bash
# Ejemplo de consulta de pacientes
curl -H "Authorization: Bearer $OCLU_API_KEY" \
     -H "Content-Type: application/json" \
     https://api.oclucrm.com/v1/patients
```

**Endpoints principales**:
- `/v1/patients` - Gestión de pacientes
- `/v1/appointments` - Agenda y citas
- `/v1/treatments` - Tratamientos y presupuestos
- `/v1/payments` - Pagos y finanzas

</Step>
</Steps>

## Personalización Avanzada

### Configuración de Campos Personalizados

Puedes agregar campos específicos para tu clínica:

```json
{
  "custom_fields": {
    "patient": [
      {
        "name": "seguro_medico",
        "type": "text",
        "label": "Seguro Médico",
        "required": false
      }
    ]
  }
}
```

### Integración con Sistemas Externos

<AccordionGroup>
  <Accordion title="Integración con Contabilidad">

    2. Exporta reportes financieros en formato compatible
    3. Sincroniza datos de pacientes y pagos

    2. Recibe notificaciones de estado

  </Accordion>

  <Accordion title="SMS y Notificaciones">
    
    Personaliza notificaciones automáticas:
    
    1. Recordatorios de citas por SMS
    2. Notificaciones de pagos pendientes
    3. Alertas de seguimiento de tratamientos
    
  </Accordion>
</AccordionGroup>

## Herramientas de Desarrollo

### Logs y Debugging

Accede a los logs del sistema para troubleshooting:

```bash
# Ver logs en tiempo real
tail -f /var/log/oclu/app.log

# Filtrar por usuario específico
grep "user_id:123" /var/log/oclu/app.log
```

### Backup y Restauración

Procedimientos para respaldo de datos:

```bash
# Backup completo de la base de datos
pg_dump oclu_production > backup_$(date +%Y%m%d).sql

# Restauración desde backup
psql oclu_production < backup_20241201.sql
```

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: "API rate limit exceeded"'>
    
    Si recibes este error:
    1. Verifica tu plan de API
    2. Implementa throttling en tus requests
    3. Contacta soporte para aumentar límites
    
  </Accordion>

  <Accordion title="Problema: Sincronización de datos">
    
    Solución:
    1. Verifica la conectividad de red
    2. Revisa los logs de webhook
    3. Ejecuta sincronización manual
    4. Contacta soporte técnico
    
  </Accordion>

  <Accordion title="Error: Permisos insuficientes">
    
    Para resolver problemas de permisos:
    1. Verifica tu rol de usuario
    2. Solicita permisos adicionales al administrador
    3. Revisa la configuración de seguridad
    
  </Accordion>
</AccordionGroup>
